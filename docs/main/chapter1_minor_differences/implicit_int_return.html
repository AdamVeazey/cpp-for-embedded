Implicit Return Type

function(int x) {
    return x + 5;
}

TL;DR

Nobody writes code like this anymore.

C++ and C standards after C89 do not support implicit return types. Functions must explicitly specify their return type.

What?

In C89, if a function didn’t specify a return type, it was implicitly treated as returning int. This behavior was a holdover from even earlier versions of C, where void wasn’t a recognized type, so all functions defaulted to returning int. However, starting with C99 and in modern C++ standards, omitting the return type results in a compile-time error, enforcing clearer and more predictable code by requiring explicit return type declarations.

C89 Example

The following is a valid C89 program

main.c
#include <stdio.h>

function(int x) {
    return x + 5;
}

int main(void) {
    int result;
    result = function(5);
    printf("result: %d\r\n", result);
    return 0;
}

Build & run with

$ gcc -std=c89 main.c && ./a.out
